01 The problem

Agents forget everything between sessions, or "solve" it by stuffing the whole transcript into context: cost grows without bound, stale facts linger forever, and contradictory facts coexist with equal confidence. The track brief names the three hard parts, and most memory demos ship only the first: efficient storage and retrieval, timely forgetting, and recall within limited context windows. Engram implements all three, and makes every decision observable.

03 The differentiator: forgetting and truth maintenance

Selection is table stakes; the two behaviors most memory agents skip are the ones you can watch live on Engram's memory board:

DECAYED
An unused low-importance episode falls below the retention floor and silently leaves recall. Preserved for audit, never deleted.
SUPERSEDED
"I just moved to Denver" retires "I live in San Diego": LLM-adjudicated, pointer to the replacement, verified live.
NOT STORED
Questions and small talk score low importance and are deliberately not stored. The board shows the decision either way.

04 Engineering rigor

  • 67 unit tests, zero network. The engine core is pure TypeScript with injected clock, embedder, and adjudicator. The budget packer carries a property-style test over 50 randomized pools: the selected set never exceeds the budget.
  • Fail-safe LLM plumbing. Every model output that feeds a decision passes a defensive parser; a parser failure can never supersede a memory (defaults: importance 0.5, no conflict, no drafts). Temperature 0 everywhere.
  • Adversarially reviewed. An independent review pass refuted 75 discrete claims in these materials against the code, tests, git history, and live probes; every finding was fixed or reworded, including a live-verified retune of the conflict-similarity gate.
  • MCP round-trip proven over the real protocol (docs/submission/mcp-roundtrip.txt): remember → recall returns the same record through the spawned stdio server.
